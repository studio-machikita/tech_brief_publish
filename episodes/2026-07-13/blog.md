🎯 **今日のアクション**
AIの安全性や生物リスクに知見を持つエンジニア・研究者は、申請フォームから参加を検討する価値があります。組織のリーダーは、自社のAI製品についても同様の外部検証プログラムの導入を検討すべきタイミングです。

## 🤖 ClaudeCodeにAIが外部サイトの閲覧・クリック・入力を可能にする内蔵ブラウザが搭載
`AI` `LLM` `Security`

<details>
<summary>📄 原題: Claude Code now has a built-in browser that lets the AI read, click, and type on external websites</summary>
</details>

> **一言で**: Claude Codeにブラウザ操作機能が追加

- Claude Codeが開発環境内でWebページを直接開き、読み取り・操作できるようになった
- 外部サイトへの書き込み操作は分類器（classifier）でスクリーニングされる
- 購入やアカウント作成などの重要操作はユーザーの承認が必要

💡 **なぜ重要か**
AIコーディングツールはこれまでコードの生成・編集が中心でしたが、開発作業には外部ドキュメントの参照やAPIの動作確認など、ブラウザ操作が欠かせません。Claude Codeにブラウザが統合されたことで、AIが開発フローをより広い範囲でカバーできるようになります。 AIエージェントが開発環境の外にある情報やサービスと直接やり取りできる時代が本格化します。一方で、AIが外部サイトに書き込む能力を持つことはセキュリティリスクも伴うため、承認フローや監査の仕組みが業界標準として整備されていくきっかけになるでしょう。

🎯 **今日のアクション**
Claude Codeのブラウザ機能を試す際は、外部サービスへの書き込み権限の範囲を事前に確認してください。特に本番環境のAPIキーや認証情報を渡さないよう、権限設計を見直すことを推奨します。

## 🤖 Codexのエージェントプロバイダーとしての活用およびJetBrains IDEにおけるエージェント機能の強化
`AI` `DevOps` `LLM`

<details>
<summary>📄 原題: Codex as agent provider and agentic enhancements in JetBrains IDEs</summary>
</details>

> **一言で**: CodexがJetBrains IDEの新エージェントに

- CodexがJetBrains IDEのエージェントプロバイダーとしてパブリックプレビュー入り
- Customizationsエディタにフック（Hooks）サポートを追加
- MCP（Model Context Protocol）サーバー管理機能が強化された
- カスタムモデルの設定サポートも新たに導入

💡 **なぜ重要か**
JetBrains IDEはJavaやKotlin開発者を中心に世界中で使われています。GitHub CopilotのエージェントにCodexが加わることで、IDE上でのAI支援コーディングの選択肢が広がります。MCP（AIモデルにツールやデータへのアクセスを提供するプロトコル）の管理機能強化は、AIエージェントが外部ツールと連携しやすくなる流れを反映しています。 IDEに直接組み込まれるAIエージェントの多様化が進むと、開発者はモデルや動作を自分の用途に合わせて細かく調整できるようになります。特定のAIプロバイダーへの依存が薄れ、エージェント選択の自由度が高まる方向に業界全体が向かうと見られています。

🎯 **今日のアクション**
JetBrains IDEを使っているチームは、Codexエージェントのパブリックプレビューを試し、既存のCopilot設定と比較評価しておくと良いでしょう。MCPサーバー管理の強化点も確認し、社内ツールとの連携可能性を検討する価値があります。

## 🤖 Amazon SageMaker HyperPodにおけるエンタープライズ推論の強化：データキャプチャ、Hugging Face、NVMe、Route 53との統合
`AI` `Cloud` `LLM`

<details>
<summary>📄 原題: Enhancing enterprise inference on Amazon SageMaker HyperPod with data capture, Hugging Face, NVMe, and Route 53 integration</summary>
</details>

> **一言で**: SageMaker HyperPodの推論機能が5つ強化

- 監査やモデル改善向けに、多段階のデータキャプチャ機能を追加
- Hugging Face Hubから直接モデルをデプロイできるように
- ローカルNVMe（高速ストレージ）活用でコールドスタートを高速化
- Route 53連携で独自ドメインのDNS設定を自動化
- カスタムサービスアカウントによるPodレベルのIAM権限管理に対応

💡 **なぜ重要か**
大規模言語モデルの推論環境では、コールドスタートの遅延（レイテンシ）やアクセス制御の細粒度化が長年の課題でした。SageMaker HyperPodはKubernetesベースのマネージド基盤として注目されており、今回の強化はエンタープライズ利用での実運用ニーズに直接応えるものです。 Hugging Face連携やNVMeキャッシュの標準化が進むことで、クラウド上でのLLM推論の運用コストと複雑さが下がると見られています。IAMのPodレベル制御はセキュリティ要件の厳しい金融・医療分野での採用を後押しする可能性があります。

🎯 **今日のアクション**
HyperPodで推論ワークロードを運用中のチームは、NVMeモデルロードとRoute 53自動DNS設定を優先的に評価してください。新規導入を検討するリーダーは、Hugging Face Hubからの直接デプロイで検証サイクルを短縮できるか確認する価値があります。

🔗 [原文を読む](https://aws.amazon.com/blogs/machine-learning/enhancing-enterprise-inference-on-amazon-sagemaker-hyperpod-with-data-capture-hugging-face-nvme-and-route-53-integration/)
